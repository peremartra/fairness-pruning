# Bias Path Figures

This directory contains the visual and tabular outputs generated by the Bias Path analysis workflow.

**Source notebook:** `notebooks/05_bias_path_analysis.ipynb`  
**Input data:** `results/neuron_analysis/{model}/{lang}/{Category}_bias_scores.json`  
**Output formats:** publication-ready `PNG` and `PDF` figures (300 dpi), plus analysis `CSV` summaries.

The content here documents where demographic-bias signals appear in model depth and neuron space, and how much overlap exists across categories and languages.

## Models and Languages Covered

| Model key | Model label | Layers | Languages |
|-----------|-------------|--------|-----------|
| `llama-3.2-1B` | Llama-3.2-1B | 16 | EN, ES |
| `llama-3.2-3B` | Llama-3.2-3B | 28 | EN, ES |
| `salamandra-2B` | Salamandra-2B | 24 | EN, ES |

Categories used in this folder: `Age`, `Gender`, `PhysicalAppearance`, `RaceEthnicity`, `Religion`.

## Figure Types

### 1) Bias Path Heatmaps (per model and language)

**Files:** `bias_path_heatmap_{model}_{lang}.png/.pdf`

**What they show**
- Layer × category heatmaps in three projection panels:
	- `gate_proj`
	- `up_proj`
	- `down_proj_input` (post-GLU signal)

**How generated in the notebook**
- `build_layer_category_matrix(...)`
- `plot_bias_path_heatmap(model_key, lang)`

**Interpretation utility**
- Locates where category-level bias signal concentrates along depth.
- Compares how projection type changes the observable bias pattern.
- Provides the base map used later for selecting candidate layers for intervention.

### 2) Cross-Model Comparison Heatmaps (relative depth)

**Files:** `bias_path_comparison_{lang}_down_proj_input.png/.pdf`

**What they show**
- Side-by-side comparison across all models using a relative-depth axis (0% to 100%).
- Uses `down_proj_input` to compare effective post-GLU bias signal consistently across architectures.

**How generated in the notebook**
- `plot_model_comparison_heatmap(lang, projection="down_proj_input")`

**Interpretation utility**
- Makes architectures with different layer counts directly comparable.
- Helps identify whether bias accumulates progressively or concentrates near final depth regions.
- Supports model-to-model comparison under a shared color scale.

### 3) Neuron Bias Heatmaps (layer × neuron-bin)

**Files:** `neuron_heatmap_{model}_{lang}_{category}_bin512.png/.pdf`

**What they show**
- Fine-grained spatial maps of neuron activity by layer and neuron-index bins.
- Three projection panels: `gate_proj`, `up_proj`, `down_proj_input`.
- Current batch generated with `bin_size=512`.

**How generated in the notebook**
- `plot_neuron_heatmap(model_key, lang, category, bin_size=512)`
- Loop over all model × language × category combinations.

**Interpretation utility**
- Moves from layer-level averages to neuron-space structure.
- Shows whether candidate neurons are spatially concentrated or distributed within layers.
- Informs whether pruning should target sparse neuron indices or broader layer segments.

### 4) Jaccard Overlap Matrices (category overlap)

**Files:** `jaccard_{model}_{lang}_{threshold}.png/.pdf`  
Thresholds in this folder: `Out01` (Top-0.1%), `top1` (Top-1%), `top5` (Top-5%).

**What they show**
- Pairwise overlap between Top-K neuron sets from different categories.
- Matrix entries are Jaccard Index values.

**How generated in the notebook**
- `extract_top_k_neurons(...)`
- `build_jaccard_matrix(...)`
- `plot_jaccard_matrix(...)`

**Interpretation utility**
- Quantifies how shared or category-specific bias circuits are.
- Supports decisions between category-specific pruning and shared-neuron strategies.
- Adds threshold sensitivity analysis (strict vs broader candidate sets).

## CSV Summaries

### `cross_lingual_overlap.csv`

**How generated**
- `cross_lingual_overlap(top_k_en, top_k_es, categories_shared)`

**Utility**
- Reports EN vs ES overlap for each category and threshold.
- Indicates transferability of candidate neurons across languages.

### `dominant_layers_summary.csv`

**How generated**
- `dominant_layers(data_norm, categories, n_layers, projection="down_proj_input", top_n=3)`

**Utility**
- Consolidates the top depth locations per category.
- Provides a compact reference for layer-targeted pruning experiments.

---

## Generation Pipeline Summary

The notebook executes the following sequence:

1. Load per-neuron bias score JSON files from `results/neuron_analysis/`.
2. Validate structure and ranges, then normalize scores per category.
3. Build layer/category matrices and render model-level heatmaps.
4. Build cross-model relative-depth comparisons.
5. Build neuron-level spatial heatmaps (layer × neuron bins).
6. Extract Top-K candidate neurons and compute Jaccard overlap matrices.
7. Compute EN/ES overlap and dominant-layer summary CSV files.
8. Save all artifacts to this folder in both `PNG` and `PDF` (figures) and `CSV` (tables).

## File Naming Conventions

| Pattern | Meaning |
|---------|---------|
| `bias_path_heatmap_{model}_{lang}` | Per-model/per-language projection heatmaps |
| `bias_path_comparison_{lang}_down_proj_input` | Cross-model comparison for one language |
| `neuron_heatmap_{model}_{lang}_{category}_bin512` | Neuron-space heatmap for one category |
| `jaccard_{model}_{lang}_{threshold}` | Category overlap matrix at one threshold |
| `cross_lingual_overlap.csv` | EN/ES overlap summary by category |
| `dominant_layers_summary.csv` | Top-layer summary by model/category |
